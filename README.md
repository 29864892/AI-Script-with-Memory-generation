A chatbot with persona management, multi-persona conversations, and memory generation. 
Memory generation and refinement were not recently tested and may not work, but the core idea was reviewing previous conversation information and generating memory objects to store in memory files based on categories. Refinement adds vector embeddings to each memory object and removes exact matches and memories with 95%+ similarity. Asks for user verification for less certain matches. The included config file should be used to edit file paths (note that some sections of the code may still have hard coded paths as this was implemented later).
Conversation Coordinator - returns an object with relevant data to decide persona turn order and apply constraints based on previous turns
ModelManagement - functions that estimate current tokens and prune context once the token limit is hit (below actual model maximum). Context is checked before AI calls. There are 2 modes to decide whether summaries are generated or old information is fully removed.
persona_management - creates a list of personas for repeated use, storing their file locations and assigning relevant system data
